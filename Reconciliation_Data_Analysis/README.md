# 数据处理标准化流程

## 项目简介

本项目用于整合运营端采集数据（Operations）和质量审核数据（QA），生成完整的数据分析报表和透视表。支持任务合并、地点映射、透视表生成等高级数据分析功能。

## 功能特点

- 自动合并多个运营端数据文件（01.csv, 02.csv, 03.csv）
- 智能匹配设备ID和日期（支持多种设备ID格式）
- 按日期+设备ID聚合QA数据（从3271条聚合为562条）
- 自动计算不合格时长占比和数据差异
- 支持多种日期格式和编码格式
- 地点信息自动匹配和补充（包含设备地点映射）
- 中文任务名称智能合并（38种任务合并为33种）
- 生成多维度透视表和统计报告
- 支持采集点×任务的交叉分析

## 使用方法

### 方式一：完整数据处理流程

1. 确保所有输入文件（01.csv, 02.csv, 03.csv, QA.csv）在同一目录下
2. 运行主脚本：

```bash
python main.py
```

3. 查看生成的 `result.csv` 文件

### 方式二：生成中文合并透视表

1. 确保已运行主脚本生成 `result.csv`

## 数据处理流程

### 步骤1: 合并Operations数据
- 读取 01.csv, 02.csv, 03.csv
- 统一列名和格式（保留5列：日期、摄像头编号、采集地点、当日采集任务、采集时长）
- 按日期排序
- 生成 Operations.csv（95行）

### 步骤2: 读取QA数据
- 读取 QA.csv（3271条原始数据）
- 自动识别编码格式

### 步骤3: 数据清洗
- 统一日期格式（YYYY-MM-DD）
- 统一设备ID格式（支持多种格式匹配）
- 转换数值类型

### 步骤4: 创建映射和聚合
- 创建设备ID到地点的映射（支持99种设备ID变体）
- 按日期+设备ID聚合Operations数据（91条）
- 按日期+设备ID聚合QA数据（119条，从3271条聚合而来）

### 步骤5: 添加地点和关联数据
- 为聚合后的QA表添加地点信息（匹配率84%）
- 关联Operations数据

### 步骤6: 构建输出表
- 计算各项指标
- 生成聚合报表

### 步骤7: 保存结果
- 保存 result.csv

## 常见问题

### Q: 为什么记录数从119条增加到562条？
A: 使用了outer join方式，现在包含三类记录：
1. 两表都有数据的记录（374条）
2. 只在Operations表的记录（66条）
3. 只在QA表的记录（113条）

### Q: 中文任务是如何合并的？
A: 通过智能规则合并相似的中文任务名称：
- 格式问题：去除多余空格
- 语义相似：如"书本收纳"→"整理书籍"
- 功能相似：如"擦拭餐具"→"用抹布擦拭碗盘"
- 英文任务保持不变

### Q: 透视表有什么用途？
A: 透视表提供多维度分析：
- 查看各采集点的任务分布
- 分析任务执行时长和频次
- 识别热点任务和地点
- 支持数据可视化和报表生成

### Q: 如何理解NOTE字段的统计？
A: NOTE字段统计显示数据质量情况：
- 合格：259条（46.09%）
- 其他因素：303条（53.91%）
- 多个NOTE用分号分隔

### Q: 数据记录差异为负数是什么意思？
A: 表示日报截屏时长大于原始上送时长，可能存在数据记录不一致的情况，需要核查。

### Q: 如何添加更多的Operations数据文件？
A: 修改 `main.py` 中的文件读取部分，添加更多文件的读取和合并逻辑。

### Q: 为什么聚合后有562条记录？
A: 使用outer join后包含了所有相关记录：
- QA表中同一天同一设备的多条记录合并为一条
- 包含仅在Operations表或仅在QA表中的记录
- 时长字段按日期+设备ID求和
