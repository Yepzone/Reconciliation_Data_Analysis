# 数据处理标准化流程

## 项目简介

本项目用于整合运营端采集数据（Operations）和质量审核数据（QA），生成按日期和设备ID聚合的完整数据分析报表。

## 功能特点

- 自动合并多个运营端数据文件（01.csv, 02.csv, 03.csv）
- 智能匹配设备ID和日期（支持多种设备ID格式）
- 按日期+设备ID聚合QA数据（从3271条聚合为119条）
- 自动计算不合格时长占比和数据差异
- 支持多种日期格式和编码格式
- 地点信息自动匹配（匹配率84%）

## 数据处理流程

### 步骤1: 合并Operations数据
- 读取 01.csv, 02.csv, 03.csv
- 统一列名和格式（保留5列：日期、摄像头编号、采集地点、当日采集任务、采集时长）
- 按日期排序
- 生成 Operations.csv（95行）

### 步骤2: 读取QA数据
- 读取 QA.csv（3271条原始数据）
- 自动识别编码格式

### 步骤3: 数据清洗
- 统一日期格式（YYYY-MM-DD）
- 统一设备ID格式（支持多种格式匹配）
- 转换数值类型

### 步骤4: 创建映射和聚合
- 创建设备ID到地点的映射（支持99种设备ID变体）
- 按日期+设备ID聚合Operations数据（91条）
- 按日期+设备ID聚合QA数据（119条，从3271条聚合而来）

### 步骤5: 添加地点和关联数据
- 为聚合后的QA表添加地点信息（匹配率84%）
- 关联Operations数据

### 步骤6: 构建输出表
- 计算各项指标
- 生成聚合报表

### 步骤7: 保存结果
- 保存 result.csv

## 输出字段说明

### result.csv（聚合报表 - 119行）

| 字段名 | 说明 | 单位 | 来源 |
|--------|------|------|------|
| 采集日期 | 数据采集日期 | - | QA表 |
| 设备ID | 摄像头设备编号 | - | QA表 |
| 采集地点 | 数据采集地点 | - | **从Operations表匹配** |
| 日报截屏上送时长（小时） | 日报记录的采集时长 | 小时 | Operations表聚合 |
| 原始上送时长（分钟） | QA原始时长 | 分钟 | QA表聚合求和 |
| 原始上送时长（小时） | 原始时长转换 | 小时 | 计算得出 |
| 运营端不合格时长（分钟） | 不合格时长 | 分钟 | QA表聚合求和 |
| 运营端不合格时长（小时） | 不合格时长转换 | 小时 | 计算得出 |
| 算法端可接受时长（分钟） | 算法可接受时长 | 分钟 | QA表聚合求和 |
| 算法端可接受时长（小时） | 算法可接受时长转换 | 小时 | 计算得出 |
| 不合格时长占比 | 不合格/原始上送 | 百分比 | 自动计算 |
| 数据记录差异 | 原始上送 - 日报截屏 | 小时 | 自动计算 |
| 任务描述 | 采集任务说明 | - | QA表 |

**重要说明：**
- 同一天同一设备的多条QA数据已聚合为一条
- 所有时长字段已按日期+设备ID求和
- 地点信息通过设备ID自动匹配

## 数据匹配规则

### 设备ID匹配
- Operations表格式：`071-b852`, `103-896a`, `23dc`
- QA表格式：`b852`, `896a`, `23dc`
- 匹配规则：支持完整匹配、前缀匹配、后缀匹配
- 匹配率：84.0%（100条有地点，19条无地点）

### 日期匹配
支持以下日期格式：
- `2026年1月31日`
- `2026/2/3`
- `2026-02-01`

统一转换为：`YYYY-MM-DD`

## 常见问题

### Q: 为什么有些记录没有地点信息？
A: 这些设备ID在Operations表（01.csv, 02.csv, 03.csv）中不存在，无法匹配地点。可以：
1. 在源文件中补充这些设备的数据
2. 手动在result.csv中填写地点信息

### Q: 数据记录差异为负数是什么意思？
A: 表示日报截屏时长大于原始上送时长，可能存在数据记录不一致的情况，需要核查。

### Q: 如何添加更多的Operations数据文件？
A: 修改 `step1_merge_operations.py` 和 `main.py` 中的步骤1，添加更多文件的读取和合并逻辑。

### Q: 为什么聚合后只有119条记录？
A: 因为QA表中同一天同一设备有多条记录，按日期+设备ID聚合后，这些记录合并为一条，时长字段求和。
